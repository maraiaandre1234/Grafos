1. Lista de Adyacencia

La lista de adyacencia representa el grafo mediante listas.
Cada nodo del grafo tiene asociada una lista que contiene todos los nodos con los que está conectado.
Esta forma es especialmente útil cuando el grafo tiene pocas aristas en comparación con la cantidad total de nodos (grafos dispersos), ya que solo se almacenan las conexiones existentes.

Ventajas:

Ocupa poco espacio.
Fácil de recorrer con algoritmos como BFS y DFS.
Ideal para grafos grandes y dispersos.

Desventajas:

Consultar si existe una arista entre dos nodos puede ser más lento.

Ejemplo:

Supongamos un grafo con conexiones:A–B, A–C, B–D.
La representación sería: 

            A: B -> C
            B: A -> D
            C: A
            D: B

2. Matriz de Adyacencia

La matriz de adyacencia usa una matriz cuadrada en la que filas y columnas representan nodos. Cada posición (i, j) indica si existe una arista entre el nodo i y el nodo j.
Se usa un 1 para indicar que existe conexión y un 0 para indicar que no hay conexión.
Esta representación es muy útil cuando se necesita consultar rápidamente si dos nodos están conectados, o cuando el grafo es denso (muchas aristas).

Ventajas:

Permite consultar si existe una arista en tiempo constante O(1).
Es más simple desde el punto de vista matemático.

Desventajas:

Consume mucho espacio: siempre es de tamaño n × n, incluso si hay pocas aristas.

Ejemplo:

Para el mismo grafo: A–B, A–C, B–D.
La matriz de adyacencia sería:

            |   | A | B | C | D |
            |---|---|---|---|---|
            | A | 0 | 1 | 1 | 0 |
            | B | 1 | 0 | 0 | 1 |
            | C | 1 | 0 | 0 | 0 |
            | D | 0 | 1 | 0 | 0 |
